# Portfolio V2 — Editorial Redesign

## Vision

This portfolio is not a showcase of React components.

It is a demonstration of engineering judgment.

A recruiter should leave thinking:

> "This person already builds like a production engineer."

Everything must reinforce the philosophy:

# Built Like Production.

---

# Design Principles

The portfolio should feel inspired by:

- Apple
- Linear
- Stripe
- Vercel
- Anthropic

Avoid:

- Portfolio templates
- Framer-style effects
- Excessive animations
- Decorative UI

Instead prioritize:

- Editorial typography
- Premium spacing
- Strong visual hierarchy
- Motion with purpose
- Simplicity
- Performance
- Accessibility

# Motion

Review every animation.

Remove unnecessary motion.

Improve:

- timing
- easing
- sequencing

Never distract.

---

# Accessibility

Minimum WCAG AA.

Support:

- keyboard navigation
- reduced motion
- screen readers
- semantic HTML
- focus states

---

# Performance

Maintain production quality.

Use:

- Server Components by default
- Lazy loading
- Optimized images
- Minimal client JavaScript

The React Bits Lanyard must not significantly degrade Lighthouse scores.

---

# Final Review

Before considering the project complete, review it as:

- Recruiter
- Hiring Manager
- Senior Engineer
- Product Designer

Identify weaknesses.

Fix them.

Repeat until no meaningful improvements remain.
