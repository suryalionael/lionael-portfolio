# CLAUDE.md

## Project Goal

Build a premium, memorable, interactive personal portfolio for **Lionael Surya**.

This is **not** a generic portfolio template.

The goal is to create a portfolio that feels like a premium software product while remaining elegant, minimal, and professional.

The portfolio should communicate craftsmanship, engineering excellence, and attention to detail.

Target audience:

- Data Engineering recruiters
- Analytics Engineering recruiters
- Software Engineering recruiters
- AI Engineering recruiters
- Hiring managers
- Technical interviewers

---

# Workflow (MANDATORY)

Before implementing ANY feature:

1. Use Serena to understand the relevant codebase.
2. Use Context7 for the latest documentation and best practices.
3. Use Higgsfield MCP for inspiration on motion design, interaction design, transitions, layouts, and premium UI ideas.
4. Follow Ponytail principles:
   - Keep implementations simple.
   - Avoid unnecessary abstractions.
   - Prefer modifying existing files.
   - Reuse components whenever possible.
5. Verify UI changes using Playwright.
6. Keep commits small, focused, and easy to review.

---

# Engineering Principles

The portfolio should feel expensive, not complicated.

Simple UI.

Sophisticated implementation.

Every interaction must have a purpose.

Avoid animations that exist only because they look cool.

Motion should guide attention.

---

# Design Philosophy

Think:

- Apple
- Linear
- Stripe
- Vercel
- Notion
- Arc Browser

DO NOT copy these products.

Instead, combine the strongest design ideas into something unique.

---

# Visual Style

Minimal.

Elegant.

Timeless.

Confident.

Professional.

Lots of whitespace.

Excellent typography.

Dark-first.

No visual clutter.

---

# Color Palette

Primary

Black (#000000)

White (#FFFFFF)

Grays only.

Accent color should be extremely subtle.

If needed:

- Electric Blue
OR
- Soft Purple

Never both.

Accent should occupy less than 10% of the interface.

---

# Typography

Use classy, modern fonts.

Preferred order:

1. Geist
2. Inter
3. Instrument Sans

Typography should carry the design.

Use generous spacing.

Large headings.

Readable body text.

---

# Motion

Motion should feel like Apple.

Use Higgsfield MCP for inspiration.

Allowed:

- Smooth page transitions
- Scroll storytelling
- Premium hover effects
- Elegant fade animations
- Layered depth
- Subtle parallax
- Card expansion
- Premium loading animations

Avoid:

- Flashy animations
- Neon effects
- Infinite rotations
- Excessive particles
- Motion sickness
- Scroll hijacking

---

# 3D

Use 3D only where it improves storytelling.

Never use 3D as decoration.

Possible uses:

- Hero depth
- Project exploration
- Architecture visualization
